# Efficiency Play: Automated CDD Engine

## Project Goal
To build a data-agnostic engine that performs automated commercial due diligence (CDD) by surfacing structural financial risks, cohort retention metrics, and market moat indicators.

## Roadmap

- [x] **Phase 1: Revenue Integrity (Audit Module)**
  - [x] Concentration Risk (HHI / 80-20 Rule)
  - [x] Cohort Retention Analysis (Vintage Analysis)
  - [x] EBITDA Normalization (One-time cost stripping)

- [x] **Phase 2: Market Moat (External Analysis Module)**
  - [x] Competitor Price Benchmarking
  - [x] Sentiment Analysis (NLP of external feedback channels)

- [x] **Phase 3: The 'Deal Room' (Dashboard Module)**
  - [x] Streamlit Integration
  - [x] Executive Summary Generator (LLM integration)

## Success Criteria
- [x] Tool identifies a 'Red Flag' in under 30 seconds from CSV input.
- [x] Visualizations are 'Partner-Ready' (Consulting-grade Plotly graphs).
- [x] Project remains strictly Python-native to avoid dependency build errors.
